# Service Butler Demo Guide

## Run the demo

Start the app in self-contained demo mode:

```bash
npm run demo
```

Demo URL: `http://127.0.0.1:3000`

Demo access:
- Open `/login`
- Click `Demo Login`

Demo mode does not require Supabase, Twilio, or any local `.env` setup for the Scanner and Weather flows.

## 3-minute walkthrough

1. Homepage value prop
   Show the marketing homepage and frame Service Butler as the ops layer that turns weather and lead signals into booked jobs.
2. Demo login
   Open `/login`, click `Demo Login`, and land on the dashboard without any auth setup.
3. Set weather service area
   Go to `Settings`, set a city/state/ZIP, and save it. The `Weather Watch` panel refreshes from the saved service area and persists after reload.
4. Run Scanner for storm response
   Open `Scanner`, confirm the service area, choose a service focus, radius, and triggers, then click `Scan Now`.
5. Convert opportunity into action
   Use `Add as Lead`, `Add as Job`, or `Claim Job`. In demo mode the action is acknowledged immediately with a clear in-app confirmation state.
