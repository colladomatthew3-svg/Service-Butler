# Autopilot Workflows

Service Butler uses Codex-powered GitHub Actions automation for routine scanning and safe issue remediation.

## Workflows

### 1) `autopilot-scan`
- **File:** `.github/workflows/autopilot-scan.yml`
- **Trigger:** nightly schedule and manual dispatch.
- **Engine:** `openai/codex-action`.
- **Purpose:** audit the repo for:
  - demo blockers,
  - UI/UX issues,
  - missing fallbacks,
  - runtime errors.
- **Output:** a GitHub issue is automatically created from the generated scan report.

### 2) `autopilot-fix`
- **File:** `.github/workflows/autopilot-fix.yml`
- **Trigger:** when an issue receives the `autopilot:fix` label.
- **Engine:** `openai/codex-action`.
- **Purpose:** read the labeled issue, implement a minimal safe fix, create a branch, and open a PR targeting `main`.
- **Guardrails:** safe demo-friendly fixes only.

## Labels

- `autopilot:scan`
  - Marks issues generated by nightly automated scanning.

- `autopilot:fix`
  - Tells the autopilot fix workflow to attempt an implementation.

- `autopilot:review`
  - Marks findings/fixes that need human validation before merge.

## PR Requirements

For any PR opened by autopilot:
- it must reference the issue it resolves,
- it must include `Closes #<issue_number>` (or equivalent closing reference),
- it should stay minimal and demo-safe.
