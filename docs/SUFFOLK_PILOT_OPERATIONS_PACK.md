# Suffolk Pilot Operations Pack

This pack contains the customer-ops artifacts for the first Suffolk restoration pilot.

## Included docs

1. [Onboarding Checklist](./SUFFOLK_ONBOARDING_CHECKLIST.md)
2. [Customer Story](./SUFFOLK_CUSTOMER_STORY.md)
3. [Lead Quality Operations](./SUFFOLK_LEAD_QUALITY_OPERATIONS.md)
4. [Lead Acceptance Reason Codes](./SUFFOLK_LEAD_ACCEPTANCE_REASON_CODES.md)
5. [Weekly Proof of Value Template](./SUFFOLK_WEEKLY_PROOF_OF_VALUE_TEMPLATE.md)
6. [Launch-Day Support and Escalation SOP](./SUFFOLK_LAUNCH_DAY_SUPPORT_ESCALATION_SOP.md)

## How to use

- Start with the onboarding checklist before any live pilot activity.
- Use the customer story as the week-one positioning guide.
- Review the lead quality operations doc to set operator expectations for week one.
- Use the reason codes the first time a lead is accepted or rejected.
- Run the proof-of-value template every week with the customer.
- Use the launch-day SOP for rollout day and any pilot incident.
